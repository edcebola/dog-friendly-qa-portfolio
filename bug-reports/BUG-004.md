# BUG-004 — Android: Place Details screen goes behind the navigation bar

**Linear ID:** DOG-38  
**Severity:** Medium  
**Priority:** Medium  
**Status:** 🔴 Open  
**Platform:** Android only  
**Component:** PlaceDetails.tsx

---

## Description

On Android devices with a visible bottom navigation bar (gesture navigation disabled), the Place Details screen content extends behind the system navigation bar. Content at the bottom of the screen is partially or fully hidden.

---

## Steps to Reproduce

1. Run the app on an Android device with the bottom navigation bar visible (not gesture mode)
2. Log in and open any venue's Place Details screen
3. Scroll to the bottom of the content

---

## Expected Result

All content is fully visible above the system navigation bar. The screen respects the bottom safe area inset.

## Actual Result

The bottom of the Place Details content (action buttons or last content block) is hidden behind the Android navigation bar. The user cannot see or interact with it.

---

## Root Cause (suspected)

The component does not apply Android-specific safe area insets at the bottom. On iOS this is handled automatically by `SafeAreaView`, but Android requires explicit handling of the navigation bar inset.

## Suggested Fix

Apply `SafeAreaView` with `edges={['bottom']}` or add a `paddingBottom` derived from `useSafeAreaInsets()` to ensure content stops above the Android navigation bar.
