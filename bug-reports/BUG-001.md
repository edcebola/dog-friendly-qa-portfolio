# BUG-001 — Report modal submit button jumps when keyboard opens

**Linear ID:** DOG-13  
**Severity:** Critical  
**Priority:** Urgent  
**Status:** Fixed ✅  
**Platform:** iOS & Android  
**Component:** PlaceDetails.tsx — report modal

---

## Description

When the user opens the report modal and taps the text input field, the keyboard causes the entire modal to compress — including the footer where the Submit button lives. The Submit button jumps upward and overlaps the content.

---

## Steps to Reproduce

1. Log in and open any venue's Place Details screen
2. Tap "Report" to open the report modal
3. Tap the text input field to trigger the keyboard
4. Observe the position of the Submit button

---

## Expected Result

The Submit button footer stays pinned at the bottom of the modal. Only the scrollable content above it compresses when the keyboard appears.

## Actual Result

The entire modal — including the footer — compresses upward. The Submit button jumps to an unexpected position mid-screen.

---

## Root Cause

`KeyboardAvoidingView` was wrapping the entire modal layout including the footer. It should only wrap the scrollable content, with the footer rendered outside it as a sibling.

## Fix Applied

Restructured the modal so `KeyboardAvoidingView` wraps only the scroll content. Footer is now a sibling element and stays pinned regardless of keyboard state.
