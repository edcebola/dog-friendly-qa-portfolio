# BUG-002 — Report confirmation alert fires 3 times

**Linear ID:** DOG-22  
**Severity:** High  
**Priority:** Urgent  
**Status:** Fixed ✅  
**Platform:** iOS & Android  
**Component:** Profile.tsx — handleReportAction

---

## Description

After submitting a report, the success alert fires three times consecutively instead of once. This gives the impression the action was taken multiple times and is confusing to the user.

---

## Steps to Reproduce

1. Log in as a regular user
2. Open the report modal from any venue's Place Details screen
3. Select any reason and tap "Submit"
4. Count how many alert popups appear

---

## Expected Result

Exactly one confirmation is shown after submitting — or the card simply disappears from the list with no alert.

## Actual Result

`Alert.alert` fires 3 times consecutively on a single submission.

---

## Root Cause

`handleReportAction` in `Profile.tsx` was calling `Alert.alert` on success. Because multiple modals were open simultaneously in memory, the handler fired once per open modal instance — resulting in 3 alerts for a single action.

## Fix Applied

Removed the `Alert.alert` call entirely. The card disappearing from the list now serves as the sole feedback, which is sufficient and less disruptive.
